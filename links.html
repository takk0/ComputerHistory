Validierung
Mit einer Validierung ist die Prüfung der Konformität von Software, beispielsweise eines Quelltextes, gemeint. So muss bei der Suchmaschinenoptimierung der HTML-Quellcode validiert werden, damit er von den Suchmaschinen-Robots korrekt erfasst wird. Die HTML-Seiten können mit einem Validator auf die korrekte Syntax hin überprüft werden.
Webstandards
Der vorliegende Artikel beschreibt die Bedeutung und ein wenig die Entwicklung von Webstandards. Ziel eines Webstandards ist eine browserübergreifende, einheitliche Implementierung, um auf verschiedenen Browsern funktionieren zu können. Zu heutiger Zeit kann man sagen, dass, aufgrund der hohen Vielfalt an Standards, die Webentwicklung komplizierter geworden ist. Durch Bereitstellen bestimmter "Best Practices" werden den Entwicklern aber Hilfen gegeben.

semantisch korrektes HTML5
Regel: Nutze Semantische HTML-Tags Eine gute Website sollte semantische HTML-Tags nutzen. Semantische HTML-Tags beschreiben bereits den darin platzierten Inhalt. Ein Absatz muss z.B. in

-Tags eingebettet sein, der für den englischen "Paragraph" steht. Der Umfang der semantsichen Tags wurde mit HTML5 stark erweitert und Webentwickler können guten Frontend-HTML-Code mit nur wenigen nicht beschreibenden und nicht semantischen Tags wie <div> oder <span> produzieren. Warum sollt eine Website semantischen HTML-Code nutzen? Google und andere Suchmaschinen lieben sie, da man den Suchmaschinen Arbeit beim Interpretieren der Websiteinhalte abnimmt, um die wesentlichen Inhalte zu finden. Suchmaschinen belohnen gutes, semantisches HTML einer Website mit einer besseren Positionierung in den Suchergebnissen.

Barrierefreies Internet
Die Internetseite "Web ohne Barrieren" bietet umfassende Informationen und Hilfen rund um das Thema Barrierfreie Informationstechnik. Auch führt die Internetseite Beispiele auf, für die Gestaltung von Barrierefreiem Internet. Um somit Diskriminierung im Internet zu reduzieren.

XML
XML ist eine Markierungssprache, oder auch Meta-Sprache. Mit ihr kann man Daten in einem Dokument strukturiert darstellen. Sie besteht lediglich aus Text. Im Gegensatz zu HTML ist es mit XML jedoch nicht möglich, Daten optisch darzustellen. Dies liegt daran, dass die Tags (Auszeichnungselemente) bei XML nicht festgelegt sind. Es besteht keine Möglichkeit, die Darstellung am Bildschirm zu beeinflussen. XML wird von vielen Programmiersprachen und Programmen unterstützt. Das macht es zu einem guten Standard, um Daten zwischen verschiedenen Medien auszutauschen.
